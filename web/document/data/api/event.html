事件

每个图表实例都可通过addEventListener(type, function)方法添加监听事件，目前支持如下几种事件：

FiCharts.event.READY

图表初始化完毕事件，实际上大多数属性接口调用都需要滞后处理，因为图表初始化完毕前所有接口都不会被真正调用，他们会被延期至初始化完毕后才正式调用。虽然图表内部会自动处理接口调用与初始化过程的关系，但建议用户最好在事件触发后再对图表进行其他操作。此事件对象属性如下：

属性 值 说明
type ready 事件类型
target 图表自身 触发此事件的对象

FiCharts.event.RENDERED

图表渲染完毕事件，分两种情况：图表加载配置/数据文件后自动渲染，渲染完毕触发此事件；调用渲染接口render()，渲染完毕触发此事件。

属性 值 说明
type rendered 事件类型
target 图表自身 触发此事件的对象

FiCharts.event.CONFIG_LOADED

配置文件加载完毕后触发此事件，仅限于setConfigFile(filePath)接口被调用后，且文件加载成功才会触发此事件。配置文件加载完毕也会同时触发rendered事件，但CONFIG_LOADED最后触发。

属性 值 说明
type configLoaded 事件类型
target 图表自身 触发此事件的对象
data configXML 配置文件

FiCharts.event.DATA_LOADED

数据文件加载完毕后触发此事件，仅限于setDataFile(filePath)接口被调用后，且文件加载成功才会触发此事件。数据文件加载完毕也会同时触发rendered事件，但DATA_LOADED最后触发。

属性 值 说明
type dataLoaded 事件类型
target 图表自身 触发此事件的对象
data dataXML 数据文件

FiCharts.event.ITEM_CLICKED

当图表中某个节点被鼠标点击后触发此事件，节点对象随事件传递出来，此事件通常用于数据钻取应用中。

属性 值 说明
type itemClick 事件类型
target 图表自身 触发此事件的对象
data 数据节点对象 数据节点对象，包含此节点上的完整信息

FiCharts.event.ITEM_OVER

当鼠标滑入图表中某个节点时触发此事件，节点对象随事件传递出来。

属性 值 说明
type itemOver 事件类型
target 图表自身 触发此事件的对象
data 数据节点对象 数据节点对象，包含此节点上的完整信息

FiCharts.event.ITEM_OUT

当鼠标滑出图表中某个节点时触发此事件，节点对象随事件传递出来。

属性 值 说明
type itemOut 事件类型
target 图表自身 触发此事件的对象
data 数据节点对象 数据节点对象，包含此节点上的完整信息

与事件相关的函数

函数 参数 说明
addEventListener type, function 添加事件，例如: chart.addEventListener('ready', function(e){...})
removeEventListener type, function 移除事件
onReady function 添加READY事件
onConfigLoaded function 添加CONFIG_LOADED事件
onDataLoaded function 添加DATA_LOADED事件
onRendered function 添加RENDERED事件
onClicked function 添加ITEM_CLICKED事件
onItemOver function 添加ITEM_OVER事件, 鼠标经滑入点时触发
onItemOut function 添加ITEM_OUT事件, 鼠标滑出节点时触发
