构造函数

图表创建从构造函数开始，目前ficharts支持两种构造函数：Pie2D和Chart2D。前者用于构建2D饼状图，后者构建除此以外的其他2D图表，但他们的属性和方法是一致的，ficharts的对外接口适用于所有图表类型。

构造函数通常只接受一个参数, 对象，其包含如下属性：

参数 取值范围 说明
id 任意字符串 必要参数，图表id, 页面加载后，以此id为标示的页面元素会被替换为图表对象，id保留。页面元素通常是div标签，也可换作其他。
width 数字(500)或者百分比('100%') 图表宽度
height 数字(500)或者百分比('100%') 图表高度
style Simple/Classic/Black 图表样式，决定图表整体视觉风格，默认为Simple
configFile 任意字符串 图表配置文件路径， 图表初始化时自动加载配置文件，并且加载成功自动渲染

例如New Chart2D({id: 'myChart', width: '100%', height: 500})定义了一个id为myChart, 宽度占满容器且高度为500像素的2D类型图表。如果构造函数仅包含一个字符串类型的参数，它既是图表id。

除此方式还有几种更简化的方式：

  • 1.仅传递图表的id，当构造函数只包含一个字符串类型的参数，并且未被逗号隔开，表示图表的id。例如：Chart2D('myChart')
  • 2.传递图表id和配置文件，当仅有的一个参数被逗号隔开为2份时，第一个是图表id，第二个是配置文件路径。例如：Chart2D("myChart,data.xml")
  • 3.传递图表id，宽高和配置文件，当仅有的一个参数被逗号隔开为4份时，依次是：图表id、宽、高、配置文件路径。例如：Chart2D("myChart,500,500,data.xml")

实例函数

图表创建好了便可以调用如下方法，操控图表行为和属性：

函数 参数 说明
setSize width, height 设置图表的宽高
setConfigXML XML 配置文件， XML格式的字符串
setDataXML XML 图表数据，XML类型的字符串
setStyle Simple/Classic/Black 图表样式，决定图表整体视觉风格。当初始化完毕后调用此方法只是改变样式配置文件， 如果需要看到效果还需要调用 render()方法
setConfigFile url 图表配置文件路径， 图表初始化时自动加载配置文件，并且加载成功自动渲染
setDataFile url 图表数据文件路径， 初始化完毕并且配置文件已经被加载后才可以调用此方法
setJsonData json json格式的图表数据
render null 渲染图表
getChartBase64Data null 获取图表截图的Base64编码数据，利用此数据可以在服务端生成图表截图，或其他文件格式。因为处于安全考虑，浏览器不允许直接调用接口方式在客户端生成文件，故开放此接口弥补此缺陷。
setCSVData [字段1，字段2] 加载CSV数据时需用到此函数，每个字段对应一列数据，字段名称需在参数中以数组格式定义。这里的字段名称与配置文件中序列的字段对应，从而序列可以据此从CSV中获取对应列的数据。
